## 「一人社長 × AI組織」という働き方

このシステムには、動画制作に必要な役割がすべて**専任のAIエージェント**として用意されています。

```
あなた（社長）
    ↓ 指示するだけ
┌───────────────────────────────────────────┐
│  AI組織                                    │
│                                           │
│  🎯 戦略  pdca-strategist                 │
│  🔍 調査  pdca-competitor / pdca-analyst  │
│  📋 企画  pdca-proposer                   │
│  📝 台本  scriptwriter                    │
│  🎬 制作  planner → script-to-remotion    │
│  🔊 品質  audio-qa                        │
│  🎞️ 出力  render-pipeline                 │
│  📤 投稿  uploader-prep                   │
└───────────────────────────────────────────┘
```

従来のYouTubeチャンネル運営では、この全役割を**一人**でやっていました。
このシステムは、それをAIエージェントに委任します。

---

## エージェント一覧と役割

システムには10種類のエージェントが含まれています。
`.claude/agents/` ディレクトリに設定ファイルが格納されています。

```
.claude/agents/
├── scriptwriter.md        # 台本生成
├── script-to-remotion.md  # 台本→Remotionコード変換
├── audio-qa.md            # 音声品質チェック
├── render-pipeline.md     # 動画レンダリング
├── uploader-prep.md       # YouTube投稿準備
├── planner.md             # 制作計画・スケジュール
├── pdca-analyst.md        # データ分析・施策評価
├── pdca-competitor.md     # 競合チャンネル調査
├── pdca-proposer.md       # 次の施策提案
└── pdca-strategist.md     # 戦略立案
```

### 各エージェントの仕事

#### scriptwriter（台本生成）

```
役割: キャラクターの掛け合い台本を生成する
入力: テーマ・ターゲット・チャンネルの方針
出力: VOICEVOX対応フォーマットの台本
```

単なる「文章を書くAI」ではありません。
- あなたのキャラクター設定（口調・関係性）を理解している
- BENS（Big/Easy/New/Safe）の4軸で自己評価する
- 冒頭フック構成（0〜9秒）・画面変化ルールを守る

#### audio-qa（音声品質チェック）

```
役割: 台本の音声生成前に問題を検出する
チェック項目:
  F1: ！？の連続（VOICEVOXが誤読する）
  F4: 不正な感情キー
  F5: 旧スピーカーIDの使用
  W1: speakTextがない英字・数字
  W2: イラストがnullのままのセリフ
```

このエージェントがあることで、「音声を生成してみたら読み間違えていた」というミスが事前に防げます。

#### pdca-competitor（競合調査）

```
役割: 競合チャンネルのバズった動画を分析する
調査内容:
  - タイトル・サムネイルのパターン
  - 視聴回数上位の動画のトレンド
  - 自チャンネルとの差別化ポイント
```

どのジャンルでも、先行チャンネルを定期的に分析することで「勝てる切り口」を見つけられます。
週1回このエージェントを走らせることが習慣化の目安です。

#### pdca-strategist（戦略立案）

```
役割: チャンネルのKPIと中期戦略を立案する
出力: 
  - 次の10本のテーマ候補
  - 検索流入を狙うキーワード
  - チャンネル目標（登録者・収益）への接続
```

## エージェントの使い方

Claude Code（Anthropic社の開発ツール）で呼び出します。

```bash
# Claude Code のインストール
npm install -g @anthropic-ai/claude-code

# プロジェクトのディレクトリで起動
cd "AI_COMPANY - 改変"
claude
```

### 台本を生成する例

Claude Codeのチャット画面で：

```
/youtube_script_specialist

テーマ: （あなたの動画テーマ）
ターゲット: （視聴者ペルソナ）
話数: 第004話
```

scriptwriterエージェントが起動し、
VOICEVOX対応の台本フォーマットで出力します。

### 競合調査をかける例

```
@pdca-competitor

今週バズった（あなたのジャンル）系YouTube動画を分析して、
来週の企画案を3つ提案してください。
```

### 品質チェックをかける例

台本を書いた後、アップロード前に：

```
@audio-qa

DialogueScene.tsx の最新エピソードのセリフをチェックしてください。
```

F（失敗）/W（警告）が表示され、問題を修正してから音声生成します。

## エージェントの設定ファイルを読む

各エージェントは `.claude/agents/` 内のMarkdownファイルで定義されています。

```markdown
---
name: audio-qa
description: |
  音声品質チェックエージェント。
  VOICEVOX生成前にSCRIPT_LINESの問題を検出する。
---

## 役割
台本データの品質問題を検出し、FAILとWARNINGで報告する。

## チェック項目
F1: ！？が連続している（例：「え！？何！？」）
...
```

**このファイルを編集することで、エージェントの振る舞いをカスタマイズできます。**

## 自分のチャンネル向けにエージェントをカスタマイズする

### キャラクター設定を変える（scriptwriter）

`scriptwriter.md` を開いて、キャラクター設定のセクションを書き換えます。

```markdown
## キャラクター設定

### 先生役（デフォルト：猫耳先生）
- 名前：（あなたのキャラ名）
- 口調：（口調の説明）
- 専門領域：（扱うジャンル）

### 生徒役（デフォルト：ケンタ）
- 名前：（あなたのキャラ名）
- 設定：（視聴者に近いペルソナ）
```

### チャンネルの方針を変える（pdca-strategist）

```markdown
## チャンネルKPI

- ターゲット登録者数：（目標数）
- 収益化の軸：（広告 / 情報商材 / 他）
- コンテンツジャンル：（例：テック解説 / 美容 / 料理）
```

## 新しいエージェントを追加する

既存の10エージェントに加えて、独自のエージェントを追加できます。

例：**サムネイル案を生成するエージェント**

```markdown
<!-- .claude/agents/thumbnail-designer.md -->
---
name: thumbnail-designer
description: |
  サムネイルのキャッチコピーと構成案を生成する。
  CTRを最大化する3要素（感情ワード・数字・ギャップ）を必ず含める。
---

## 役割
動画タイトルを受け取り、以下を出力する：
1. サムネイルのキャッチコピー案（3案）
2. 画面構成（テキスト配置・色・キャラ表情）
3. NGチェック（240px縮小時に文字が読めるか）
```

作成したら `.claude/agents/` に置くだけで、すぐに `@thumbnail-designer` で呼べます。

## なぜ「エージェントの役割分担」が重要なのか

### 文脈の切り離し

一つのAIに「調査も台本も品質チェックも」やらせると、前の会話が邪魔をします。
エージェントを分けることで、各タスクが**クリーンな文脈**で実行されます。

### 再現性が高い

「scriptwriterはいつも同じルールで台本を書く」という一貫性が生まれます。
人間のように「今日は気分が乗らないから質が低い」がありません。

### 並列実行できる

競合調査・素材収集・前の動画の分析を**同時に走らせる**ことができます。
人間1人では絶対にできない速度で情報処理が進みます。

## AI組織で「仕組みを作る」という考え方

このシステムの本質は「コード」ではなく**「組織設計」**です。

```
❌ 普通の考え方：
「AIに仕事を手伝ってもらう」
→ その都度、プロンプトを書く
→ 品質がバラつく
→ 再現できない

✅ このシステムの考え方：
「AIに役割を与えて組織を作る」
→ エージェントが職人として機能する
→ 品質が安定する
→ 自分がいなくても動く
```

FIRE達成の本質は「**自分がいなくても動く仕組みを作ること**」です。
このAI組織は、YouTube事業をそのような仕組みに変えるための基盤です。
